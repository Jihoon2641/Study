/* DOM Tree에는 존재 */

/*
  DOM Tree:    html > body > div.box > p.hidden   ← 있음
  CSSOM Tree:  .hidden { display: none }          ← 있음
  Render Tree: p.hidden                           ← ❌ 제외!
  
  display:none = "화면에 공간 자체를 차지하지 않음"
  visibility:hidden = Render Tree에 포함되지만 투명 (공간 차지)
*/
브라우저 렌더링 전체 파이프라인 (6단계)
1
① HTML Parsing → DOM Tree 생성
브라우저가 HTML 파일을 위에서 아래로 읽으며 노드를 만들고 트리를 구성합니다.
태그를 만나면 노드 생성 → 부모/자식 관계를 파악 → 트리 완성
<div> → HTMLDivElement 노드 생성
2
② CSS Parsing → CSSOM Tree 생성
<style> 태그 또는 .css 파일을 파싱해서 스타일 규칙 트리를 만듭니다.
이 단계는 HTML 파싱과 병렬로 진행됩니다.
h1 { color: green } → h1 노드에 스타일 연결
3
③ Render Tree 생성
DOM + CSSOM을 합쳐서 실제로 화면에 보여야 할 노드만 남긴 트리를 만듭니다.
head, title, display:none은 이 트리에서 제외됩니다.
DOM ⊕ CSSOM = Render Tree
4
④ Layout (Reflow)
각 노드의 정확한 위치(x,y)와 크기(width, height)를 계산합니다.
뷰포트 크기, 폰트 크기, padding/margin 등을 모두 고려합니다.
div.box → x:0, y:0, w:800px, h:120px
5
⑤ Paint (Rasterize)
Layout에서 계산된 정보를 바탕으로 실제 픽셀을 화면에 그립니다.
텍스트, 색상, 이미지, 그림자, 테두리를 레이어로 그립니다.
h1 → 픽셀로 초록색 텍스트 렌더링
6
⑥ Composite
여러 레이어를 합성해서 최종 화면을 완성합니다.
GPU가 이 과정에 참여합니다. ✅ 이 단계 완료 후 화면에 보입니다!
레이어1 + 레이어2 + ... = 최종 화면 🖥️
JS로 DOM 변경 시 — Reflow vs Repaint
JavaScript 
// ❗ Reflow 발생 — 크기/위치가 바뀌어 Layout부터 다시 계산
document.getElementById('box').style.width = '200px';
//                                ④ Layout → ⑤ Paint → ⑥ Composite

// ❗ Repaint만 발생 — 위치는 그대로, 색깔만 바뀜
document.getElementById('box').style.color = 'red';
//                                ⑤ Paint → ⑥ Composite (Layout 생략)

// ✅ 가장 빠름 — transform은 Composite 단계만 사용
document.getElementById('box').style.transform = 'translateX(10px)';
//                                ⑥ Composite만 (GPU 처리)
React 예제 코드
JSX (React) 
import { useState } from 'react';

function Counter() {
  // ① State 선언: count = 0
  const [count, setCount] = useState(0);

  function increase() {
    setCount(count + 1);  // ② State 변경 → 리렌더링 트리거
  }

  // ③ JSX 반환 = Virtual DOM 설계도
  return (
    <div className="box">
      <h1>숫자: <span>{count}</span></h1>
      <button onClick={increase}>+1 증가</button>
    </div>
  );
}

/*
  JSX는 사실 이렇게 변환됩니다 (Babel):
  React.createElement('div', {className:'box'},
    React.createElement('h1', null, '숫자: ',
      React.createElement('span', null, count)
    ),
    React.createElement('button', {onClick: increase}, '+1 증가')
  )
*/
Virtual DOM Diffing — 바뀐 부분만 찾아서 업데이트
🔵 이전 Virtual DOM (count = 0)
div.box
└ h1
├ "숫자: "
└ span
└ "0"
└ button
└ "+1 증가"
⇒
🔴 새 Virtual DOM (count = 1)
div.box
└ h1
├ "숫자: "
└ span
└ "1" 변경!
└ button
└ "+1 증가"
🔍 Diffing 결과 → span 안의 텍스트 노드 "0" → "1" 만 변경됨
⚡ Reconciliation → 실제 DOM에서 해당 텍스트 노드 하나만 업데이트
🚫 나머지 div, h1, button은 건드리지 않음 = 성능 최적화
React 렌더링 전체 흐름
1
JSX → React.createElement()
Babel이 JSX를 브라우저가 이해하는 JS 함수 호출로 변환합니다.
<h1> → React.createElement('h1', null, ...)
2
Virtual DOM 생성 (메모리)
React가 메모리 안에 가상의 DOM 트리를 만듭니다. 실제 DOM보다 훨씬 가볍습니다.
{ type:'div', props:{...}, children:[...] }
3
최초: 실제 DOM에 마운트
처음 렌더링 시, Virtual DOM을 실제 DOM으로 변환해서 브라우저에 삽입합니다.
ReactDOM.createRoot(root).render(<App/>)
4
State 변경 → 리렌더링
setCount() 호출 → 컴포넌트 함수 재실행 → 새 Virtual DOM 생성됩니다.
setCount(1) → Counter() 재실행
5
Diffing (비교)
이전 Virtual DOM과 새 Virtual DOM을 비교해서 변경된 노드를 찾습니다.
span "0" ≠ span "1" → 변경 감지
6
Reconciliation → 최소 업데이트
변경된 부분만 실제 DOM에 반영합니다. 브라우저의 Paint/Reflow를 최소화합니다.
span.textContent = "1" ← 이것만!
