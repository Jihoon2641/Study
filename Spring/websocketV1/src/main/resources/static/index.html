Short Polling의 근본적 문제

클라이언트는 새 메시지가 있는지 알 수 없기 때문에
무조건 1초마다 서버에 물어봅니다.

채팅이 조용한 동안에도 HTTP 연결 수립 → 요청 → 응답 사이클이 반복됩니다.

사용자 100명이 채팅방에 있으면?
→ 초당 100개의 HTTP 요청 발생
→ 대부분은 "새 메시지 없음" 응답
→ 서버 자원 낭비, 메시지 딜레이 최대 1초
