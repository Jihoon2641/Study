Raw WebSocket의 한계
지금 직접 구현한 것들

① 프로토콜 설계
type: CHAT | JOIN | LEAVE | PING...
메시지 구분 규약을 직접 만들었습니다.

② 방(Room) 관리
sessionId → roomId 맵핑을 직접 관리.
STOMP라면 SUBSCRIBE /topic/room-1 한 줄.

③ 브로드캐스트
방의 세션 목록을 순회하며 직접 전송.
STOMP라면 convertAndSend() 한 줄.

④ Dead Connection 감지
스케줄러로 30초 Ping 없으면 직접 제거.

결론: 이 모든 것이
보일러플레이트 코드입니다.
