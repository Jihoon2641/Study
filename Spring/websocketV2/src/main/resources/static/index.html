Short Polling과의 결정적 차이

서버는 새 메시지가 없으면 응답을 보내지 않고 연결을 붙잡아 둡니다.

메시지가 오는 순간 대기 중인 모든 클라이언트에게 즉시 Push합니다.

개선된 점:
→ 빈 응답 폭발 제거
→ 메시지 수신 지연 거의 0ms

여전히 남은 문제:
→ 타임아웃마다 새 HTTP 연결 수립
→ 서버 스레드를 연결당 1개씩 점유
→ 클라이언트가 많으면 스레드 부족
